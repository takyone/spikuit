# Python Code Extractor

Triggered by `spkt-ingest` when the input is a `.py` file, a directory of
`.py` files, or a `pyproject.toml`.

## When to use

- Single Python source file (`.py`)
- Python package directory (walk `*.py` recursively)
- `pyproject.toml` (extract package metadata + summary neuron, then walk
  the package source)

## Preprocessing

For each file, list top-level definitions with the standard library:

```bash
python3 -c '
import ast, json, sys
tree = ast.parse(open(sys.argv[1]).read())
items = []
for node in tree.body:
    if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef)):
        items.append({"kind": "function", "name": node.name, "line": node.lineno,
                      "doc": ast.get_docstring(node)})
    elif isinstance(node, ast.ClassDef):
        items.append({"kind": "class", "name": node.name, "line": node.lineno,
                      "doc": ast.get_docstring(node),
                      "methods": [n.name for n in node.body
                                  if isinstance(n, (ast.FunctionDef, ast.AsyncFunctionDef))]})
imports = []
for node in ast.walk(tree):
    if isinstance(node, ast.Import):
        imports += [a.name for a in node.names]
    elif isinstance(node, ast.ImportFrom) and node.module:
        imports.append(node.module)
print(json.dumps({"items": items, "imports": imports}))
' <file.py>
```

(Inline the snippet — no helper scripts shipped, so it works in any
environment with Python 3.11+.)

## Neuron creation rules

| Definition | Neuron shape |
|---|---|
| Top-level function | `# <module>.<func>()\n\n<docstring>\n\nSignature: ...` |
| Top-level class | `# <module>.<Class>\n\n<docstring>\n\nMethods: ...` |
| Module without top-level defs | One neuron summarizing the module's purpose |
| `pyproject.toml` | One neuron with name, version, description, deps |

Use `type: code`, `domain: <package-name>`. Attach `--source-url file://<abs-path>`.

Skip:
- Functions/classes whose name starts with `_` (private) unless they have a docstring
- Files under `tests/` and `__pycache__/`
- Files generated by build tools

## Synapse suggestions

After creating neurons, walk the import graph:

1. For each `from X import Y`, if `X` corresponds to a neuron we just created
   in the same package, add: `spkt synapse add <importer> <imported> -t requires`
2. For class methods that call sibling methods on `self`, add `relates_to`
   between the class neuron and any helper neurons it depends on
3. Skip stdlib and third-party imports — only intra-package synapses

## Output

```
Added N neurons (M functions, K classes), L synapses from imports (python-code extractor).
```
