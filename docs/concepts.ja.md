# コンセプト

## 背景: なぜニューラルナレッジグラフなのか？

Spikuitの設計は3つの分野から着想を得ています:
**計算論的神経科学**、**認知/発達心理学**、**グラフベース機械学習**。
このセクションでは背景にある理論と、Spikuitがそれらをどう適用しているかを説明します。

### 神経科学から

#### ニューロンと活動電位

生体のニューロンは**活動電位**（スパイク）と呼ばれる離散的な電気パルスで
通信します。ニューロンは近傍からの入力を蓄積し、閾値を超えると発火して
下流のニューロンに信号を送ります。これは二値的な全か無かの現象です。

Spikuitでは**Spike**が復習イベントです。概念を復習する（スパイクを発火する）と、
信号がつながった知識に伝播します -- ちょうど活動電位が神経回路を伝播するように。

#### シナプス可塑性とヘッブ学習

「一緒に発火するニューロンは結びつく」（Hebb, 1949）。
ニューロン間の接続は、時間的に近い活性化により強化されます。
これが連合学習の生物学的基盤です。

Spikuitはこれを**STDP**（後述）で実装しています:
時間窓の中で関連概念を復習すると、それらの接続が強化されます。

#### スパイクタイミング依存可塑性（STDP）

STDPはヘッブの法則を時間的非対称性で精緻化します:

- **Preが先に発火**（因果的順序）: 接続が強化される（**LTP**: 長期増強）
- **Postが先に発火**（逆順序）: 接続が弱化される（**LTD**: 長期抑圧）

変化量は時間差に対して指数関数的に減衰します。これにより方向性のある
学習が生まれます: AをBより先に復習し続けると、A→B接続が強化される一方、
B→Aは弱まる可能性があります。

Spikuitでは `tau_stdp` 日（デフォルト: 7）以内の共発火タイミングで
エッジ重みが更新されます。「Functor」の後に「Monad」を同じ日に
学習すると、接続が自然に強化されます。

#### 漏れ積分発火モデル（LIF）

LIFモデルは、ニューロンが入力を蓄積（積分）しながら徐々に電荷を
失う（漏れ）様子を記述します。近傍からの入力が膜電位を積み上げ、
閾値を超えるとニューロンが発火し、その後リセットされます。

Spikuitでは**復習圧力**にLIFを使います: 近傍の復習が圧力を押し上げ、
時間が圧力を減衰させます。高圧力のニューロンは復習の「準備ができている」
-- 関連する学習によってこの概念が活性化されていることをシステムが示しています。

#### 拡散活性化

記憶の中で概念が活性化されると、連合リンクを通じて関連概念に活性化が
拡散します（Collins & Loftus, 1975）。「犬」を考えると「猫」や「骨」が
プライミングされるが「代数」はされない、という現象を説明します。

Spikuitでは**APPNP**（Personalized PageRank）で拡散活性化を実装:
概念を復習すると、接続強度で重み付けされたグラフ近傍に活性化が送られます。

### 認知心理学・発達心理学から

#### 忘却曲線と間隔反復

エビングハウス（1885）は、記憶が時間とともに指数関数的に減衰することを
示しました。しかし検索に成功するたびに記憶痕跡が強化され、将来の減衰が
遅くなります。忘れる直前に復習する -- 最適な復習タイミング -- が記憶定着
の効率を最大化します。

Spikuitはニューロン単位のスケジューリングに**FSRS v6**（Free Spaced
Repetition Scheduler）を使用します。FSRSは各アイテムの**stability**
（90%想起確率までの時間）と**difficulty**をモデル化し、復習のたびに
更新します。

#### テスティング効果

能動的に記憶から情報を検索することは、再読よりも記憶を強化します
（Roediger & Karpicke, 2006）。失敗した検索試行でさえ、後の想起を
改善します。

これがSpikuitのLearnプロトコルが「提示→評価」の構造を取る理由です。
単にコンテンツを見せるのではなく、Scaffoldシステムが表示量を制御し、
習熟度が上がるにつれてサポートを段階的に減らしていきます。

#### 発達の最近接領域（ZPD）とスキャフォールディング

ヴィゴツキー（1978）はZPDを、学習者が独力でできることと、
支援があればできることのギャップとして記述しました。
**スキャフォールディング**（Wood, Bruner & Ross, 1976）は
学習者がZPD内で活動できるよう助ける一時的なサポートで、
能力が向上するにつれて段階的に撤去されます。

SpikuitのScaffoldシステムはFSRS状態からサポートレベルを算出します:

| レベル | FSRS状態 | サポート |
|--------|---------|---------|
| **FULL** | Learning（新規） | フルコンテンツ、最大ヒント、易しい問題 |
| **GUIDED** | Relearning / 低stability | 部分コンテンツ、ヒント利用可 |
| **MINIMAL** | Review、中程度のstability | タイトルのみ、難しい問題 |
| **NONE** | Review、高stability | 純粋な想起、応用レベル |

さらに**Context**（スキャフォールディング素材となる強い近傍）と
**Gaps**（先に学習すべき弱い前提条件）を特定します。

#### スキーマ理論

スキーマは知識を組織化する心的枠組みです（Bartlett, 1932; Piaget）。
新しい情報は既存のスキーマに接続できると学習しやすくなります。
この原理は**同化**と呼ばれます。

Spikuitのナレッジグラフは本質的にスキーマです。新しい概念を追加すると、
`LearnSession.ingest()` が自動的に関連する既存知識を検索し、
新情報を既存のスキーマに接続しやすくします。

### グラフベース機械学習から

#### PageRankとAPPNP

GoogleのPageRank（Page et al., 1999）はリンクグラフの構造からWebページの
スコアを計算します。APPNP（Gasteiger et al., 2019）はこれをニューラル
ネットワークに適用し、テレポート確率を持つPersonalized PageRankで
伝播をローカルに保ちます。

Spikuitでは2つの目的でAPPNPを使用:

1. **拡散活性化**: 1ノード復習 → 近傍に活性化が流れる
2. **検索スコアリング**: 中心性（概念のつながり度合い）が検索ランキングに寄与

## Spikuitのアルゴリズム

### FSRS (Free Spaced Repetition Scheduler)

ニューロン単位の間隔反復。各ニューロンはFSRS Cardを持ち、stability、
difficulty、次回復習日を追跡します。伝播はFSRS状態に**一切触れません**
-- 影響するのは圧力だけです。

### APPNP (Approximate Personalized Propagation of Neural Predictions)

スパイクを発火すると、Personalized PageRankを通じて近傍に活性化が伝播します。
これが関連概念に**復習圧力**を生み出し、次に何を学ぶべきかを示唆します。

```
Z = (1 - alpha) * A_hat @ Z + alpha * H
```

- `alpha` = テレポート確率（高いほどローカル）
- `A_hat` = 自己ループ付き正規化隣接行列
- `H` = 初期活性化（グレードに依存した強度）

### STDP (Spike-Timing-Dependent Plasticity)

`tau_stdp`日以内の共発火タイミングに基づいてエッジ重みが更新されます:

- **Preが先に発火（LTP）**: `dw = +a_plus * exp(-|dt| / tau)`
- **Postが先に発火（LTD）**: `dw = -a_minus * exp(-|dt| / tau)`

関連概念を一緒に復習すると接続が強化され、しないと弱まります。

### LIF (Leaky Integrate-and-Fire)

復習圧力は近傍の発火から蓄積し、指数関数的に減衰します:

```
pressure(t) = pressure * exp(-dt / tau_m)
```

圧力が閾値を超えると、ニューロンは自発的復習の「準備ができた」状態になります。

## セッション

セッションはBrain（Circuit）のインタラクションモードです。
Brainは共通のバックエンド、セッションがインタラクション方法を定義します。

### QABotSession

自己最適化するRAGチャット:

- **ネガティブフィードバック**: 類似フォローアップクエリが前回結果をペナルティ
- **Accept**: 明示的な正のフィードバックでニューロンをブースト
- **重複排除**: 既に返されたニューロンを除外
- **永続/一時**: ブーストのコミット有無を選択可能

### LearnSession

会話型ナレッジキュレーション:

- **ingest**: ニューロンを追加し、関連概念を自動発見
- **relate**: シナプスを作成または強化
- **search**: グラフ重み付き検索
- **merge**: 重複ニューロンを統合（シナプス転送 + コンテンツ結合）

### 会話型RAGキュレーション

核心的な洞察: **会話が直接検索品質を改善する**。

従来のRAGはナレッジベースを静的に扱います。Spikuitのグラフは生きています
-- 復習、検索結果の受諾、キュレーション操作のすべてが構造を改善します。
結果として、**使うほど良くなる**RAGシステムが生まれます。

## Embedder

差し替え可能なテキスト埋め込み（複数プロバイダー対応）:

| プロバイダー | API | ユースケース |
|-------------|-----|-------------|
| `openai-compat` | `/v1/embeddings` | LM Studio, Ollama /v1, vLLM, OpenAI |
| `ollama` | `/api/embed` | Ollamaネイティブ API |
| `none` | -- | 埋め込みなし（キーワード検索のみ） |

埋め込みはsqlite-vecにKNN検索用に保存され、retrieveのスコアリングで使用されます:

```
score = max(キーワード類似度, 意味的類似度) * (1 + 検索可能性 + 中心性 + 圧力 + ブースト)
```

## Scaffold

ZPD着想のサポートレベル。FSRS状態とグラフ近傍から算出:

| レベル | 条件 | 内容 |
|--------|------|------|
| **FULL** | 新規 / Learning状態 | 最大ヒント、フルコンテンツ、易しい問題 |
| **GUIDED** | Relearning / 低stability | リクエスト時ヒント、部分コンテンツ |
| **MINIMAL** | Review + 中程度のstability | 難しい問題、タイトルのみ |
| **NONE** | 高stability（習得済み） | 純粋な想起、応用レベル |

Scaffoldはさらに以下を特定します:

- **Context**: 強い近傍（学習者がよく知っているスキャフォールディング素材）
- **Gaps**: 弱い前提条件（先に学習すべきもの）

## Learnプロトコル

抽象プロトコル: select -> scaffold -> present -> evaluate -> record

- **Flashcard**: セルフグレードのフラッシュカード（LLM不要）。
  Scaffoldレベルがコンテンツの表示量を制御。
- **Quiz**（agents経由）: LLM生成の問題、ニューロン単位のグレーディング。
