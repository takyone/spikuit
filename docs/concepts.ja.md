# コンセプト

## Brain

**Brain**は自己完結型のナレッジスペースです —
[Obsidianのvault](https://obsidian.md/)やgitリポジトリのようなものです。
各Brainは`.spikuit/`ディレクトリに格納され、独自のナレッジグラフ、
設定、復習スケジュールを持ちます。

```
my-project/
└── .spikuit/
    ├── config.toml    # Brain設定（名前、エンベッダー）
    ├── circuit.db     # SQLiteデータベース
    └── cache/         # エンベディングキャッシュ
```

### 複数のBrain

用途やドメインごとに、好きなだけBrainを作れます。

```bash
~/math/.spikuit/      # 圏論、代数
~/french/.spikuit/    # フランス語の語彙と文法
~/work/.spikuit/      # 仕事関連の知識
```

### 自動検出

gitと同じように、`spkt`はカレントディレクトリから上方向に`.spikuit/`を探します。
別のBrainを操作するには`--brain <path>`を使用します。

## ナレッジグラフ

Spikuitは知識を**グラフ**として整理します：コンセプトがノード、
関係がエッジです。

### Neuron

**Neuron**は知識の最小単位 — Markdownで保存されます。
各Neuronにはtype（concept、term、procedureなど）と
domain（math、french、csなど）を設定できます。

```bash
spkt add "# Functor\n\n圏の間の構造を保つ写像。" -t concept -d math
```

### Synapse

**Synapse**は2つのNeuron間の型付き接続です。

| タイプ | 方向 | 意味 |
|-------|------|------|
| `requires` | A → B | AはBの理解が前提 |
| `extends` | A → B | AはBを拡張する |
| `contrasts` | A ↔ B | AとBは対比関係 |
| `relates_to` | A ↔ B | 一般的な関連 |

接続には重みがあり、使い方に応じて**強くなったり弱くなったり**します。
関連するコンセプトを近い時期に復習すると、接続が強くなります。

### ソース（Source）

**Source**は知識の出典を追跡します — URL、論文、書籍、ファイルなど。
ソースにより回答時の引用とバージョン追跡が可能になります。

```bash
spkt add "# 重要な発見" --source-url "https://paper.com" --source-title "論文"
spkt learn "https://paper.com" -d cs --json    # 一括取り込み
```

1つのソースから複数のNeuronが生成できます（1:N）。
複数のNeuronが同じソースを共有できます（M:N）。URLで重複排除されます。

### コミュニティ

Spikuitは**コミュニティ** — 密に接続されたNeuronのクラスタ — を
Louvainアルゴリズムで検出します。コミュニティは検索を改善し、
トップヒットと同じクラスタの結果をブーストします。

```bash
spkt communities --detect          # コミュニティを検出
spkt communities --json            # 現在の割り当てを表示
```

コミュニティは**可視化**にも使われ、ノードがコミュニティごとに
色分けされるため、知識クラスタの視覚的な識別が容易になります。

### なぜグラフなのか？

フラッシュカードのデッキは各カードを独立に扱います。しかし知識は
独立していない — 「モナド」を理解するには「ファンクター」の理解が
先に必要です。グラフはこれらの関係を捉え、以下を可能にします：

- **前提知識の検出**: 何を先に学ぶべきかが分かる
- **活性化の伝播**: 一つの復習が関連コンセプトに波及
- **賢い検索**: テキスト類似度だけでなく、グラフ構造も考慮したランキング

## 間隔反復

各Neuronは[FSRS](https://github.com/open-spaced-repetition/fsrs4anki)による
独自の復習スケジュールを持ちます — モダンなAnkiと同じアルゴリズムです。

| Grade | 意味 |
|-------|------|
| `miss` | 思い出せなかった |
| `weak` | 不確か |
| `fire` | 正解 |
| `strong` | 完璧な想起 |

復習すると2つのことが起きます：

1. **スケジュールが更新** — 正解で安定性が上がり、失敗で下がる
2. **関連コンセプトが影響を受ける** — 「Functor」の復習が「Monad」の
   復習タイミングをわずかに早める

つまり復習キューは個別の期日だけでなく、知識の*構造*に影響されます。

## 検索

Spikuitの検索は複数のシグナルを組み合わせます：

```
関連度 = テキスト類似度 × (1 + 記憶の強さ + 中心性 + 圧力 + フィードバック + コミュニティブースト)
```

- **テキスト類似度**: キーワード + セマンティック（エンベディング）
- **記憶の強さ**: よく覚えているコンセプトが上位に
- **中心性**: よく接続されたコンセプトが上位に
- **圧力**: 最近の復習で「プライミング」されたコンセプトが上位に
- **フィードバック**: 過去の検索フィードバック（承認/不採用）がランキングを調整
- **コミュニティブースト**: トップヒットと同じコミュニティの結果をブースト

同じクエリでも、知識や使用パターンの変化に応じて結果が変わります。

## スキャフォールディング

各コンセプトの理解度に応じて適応します：

| レベル | いつ | 意味 |
|-------|------|------|
| Full | 新しいコンセプト | 最大サポート — 全コンテキスト、簡単な問題 |
| Guided | 学習中 | 一部サポート — ヒント利用可能、適度な難易度 |
| Minimal | 定着してきた | サポート少なめ — 難しい問題 |
| None | 習得済み | 記憶から想起 — 応用レベルの課題 |

**ギャップ**（まだ習得していない前提知識）も検出し、
先に復習することを提案します。

## セッション

BrainのLLM駆動インタラクションモード：

| セッション | 機能 |
|-----------|------|
| **QABotSession** | RAGチャット — 知識ベースに質問して回答を得る。フィードバックで検索品質が向上。 |
| **LearnSession** | 会話で知識を追加 — 関連の自動発見、重複検出。 |
| **TutorSession** | AIチューター — ギャップを診断し、概念を教え、クイズし、フィードバック。 |

セッションは**永続的**（フィードバックが将来に活用）または
**一時的**（セッション終了で破棄）にできます。

## アーキテクチャ

```
spikuit-core/     # 純粋エンジン（LLM依存なし）
├── Circuit       #   ナレッジグラフ + FSRS + 伝播
├── Embedder      #   プラガブルなテキストエンベディング
├── Sessions      #   QABot, Learn, Tutor
└── Learn         #   クイズ戦略（Flashcard, AutoQuiz）

spikuit-cli/      # spktコマンド（Typer）
spikuit-agents/   # エージェントスキルとアダプター
```

コアエンジンは**LLM非依存** — すべての`spkt`コマンドはLLMなしで動作します。
セッションとエージェントスキルがLLM駆動のインタラクションを追加します。

アルゴリズムの詳細（FSRS、グラフ伝播、スコアリング式）は
[付録: アルゴリズム](appendix.ja.md)を参照してください。
